# 验证

用于定义项目期望的验证流程。

## 标准验证

- [ ] 如果存在 lint 命令，lint 通过。
- [ ] 相关测试通过。
- [ ] 如果存在构建命令，构建通过。

## 必要证据

- [ ] 记录每个已执行的命令或手动检查。
- [ ] 记录每个检查的结果。
- [ ] 说明每个跳过的检查。
- [ ] 验证缺失或失败时，不要标记工作完成，除非风险已明确记录。

## 项目专属验证

- [ ] 执行从项目中发现的领域专属检查。
- [ ] 当命令检查不足时，执行手动工作流验证。
- [ ] 记录跳过检查的原因。

## 记录位置

把命令、结果、失败和跳过项写入当前 run 的 `execution-log.md`。
