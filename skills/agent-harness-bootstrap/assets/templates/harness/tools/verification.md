# Verification

Use this file to define the expected verification process.

## Standard Verification

- [ ] Lint command passes, if available.
- [ ] Relevant tests pass.
- [ ] Build command passes, if available.

## Browser Verification

For browser projects:

- [ ] Open the relevant page in a real browser.
- [ ] Check console errors.
- [ ] Check failed network requests.
- [ ] Verify the target workflow manually.
- [ ] Check mobile and desktop layouts when UI changed.

## Recording Results

Write commands, results, failures, and skipped checks in the active run's `execution-log.md`.
