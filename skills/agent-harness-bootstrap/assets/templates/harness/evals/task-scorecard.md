# Task Scorecard

## Spec Compliance

- [ ] All acceptance criteria are complete.
- [ ] Out-of-scope items were not implemented.

## Code Quality

- [ ] Changes follow local conventions.
- [ ] Complexity is appropriate for the task.
- [ ] No unrelated refactors were introduced.

## Verification

- [ ] Lint passed or was not applicable.
- [ ] Tests passed or skipped checks are explained.
- [ ] Build passed or was not applicable.
- [ ] Project-specific verification passed or skipped checks are explained.
- [ ] Completion is backed by recorded evidence, not only by agent summary.

## Risk

- [ ] Residual risks are documented.
- [ ] Follow-up work is identified when needed.

## Harness Feedback

- [ ] Missing or stale harness context was captured in `Context Updates` or `evaluation.md`.
- [ ] Repeated mistakes or unclear rules were proposed as harness updates.
