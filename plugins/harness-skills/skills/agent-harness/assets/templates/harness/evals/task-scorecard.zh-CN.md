# 任务评分表

## 规格符合度

- [ ] 所有验收标准已完成。
- [ ] 未实现范围外事项。

## 代码质量

- [ ] 修改符合本项目约定。
- [ ] 复杂度与任务规模匹配。
- [ ] 根组件、页面组件、handler、service 和模块职责保持聚焦。
- [ ] 非平凡前端功能已合理分离状态、数据访问和 UI 渲染。
- [ ] 后端修改包含必要的错误处理和日志/可观测性。
- [ ] 非显而易见的业务规则、边界条件和权衡已用有用注释说明。
- [ ] 没有引入无关重构。

## 验证

- [ ] Lint 通过，或不适用。
- [ ] 测试通过，或已说明跳过原因。
- [ ] 构建通过，或不适用。
- [ ] 项目专属验证通过，或已说明跳过原因。
- [ ] 完成结论有记录证据支撑，而不只是 Agent 摘要。
- [ ] 应用代码变更已执行代码质量 review，或已记录跳过原因。

## 风险

- [ ] 残留风险已记录。
- [ ] 必要的后续工作已识别。

## Harness 反馈

- [ ] 缺失或过期的 harness 上下文已记录到 `Context Updates` 或 `evaluation.md`。
- [ ] 重复错误或不清晰规则已作为 harness 更新建议提出。
